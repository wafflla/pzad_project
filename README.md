# ИИ-ассистент для написания постов в социальные сети приютов для животных

Волонтер загружает фотографию животного и при необходимости добавляет краткие текстовые заметки о его имене, возрасте, характере или особенностях. На основе этих данных система автоматически генерирует готовый текст поста.
Итоговый текст не только описывает животное, но и соответствует стилю конкретного приюта за счет использования ранее опубликованных постов.

В рамках проекта выполнены следующие задачи:
1. Сбор и подготовка данных приютов
2. Извлечение описания собаки из изображения
3. Учет стилистики других текстов приюта
4. Генерация итогового текста

## Сбор и подготовка данных приютов

На данном этапе проекта была выполнена первичная сборка корпуса данных о животных из приютов. Были спарсены данные с шести источников: Красная Сосна, Юна, Искра, Некрасовка, Печатники Pets и Бирюлево. Цель этого этапа — сформировать единую базу карточек животных в семи-едином формате (т.к. у каждого сайта отличается информация, хранящаяся в карточках и формат). Базав дальнейшем будет использоваться для анализа описаний LLMкой и генерации новых текстов для карточек.

Для каждого сайта парсинг выполнялся в два шага. Сначала собирались ссылки на все доступные карточки животных из каталога или списка объявлений. После этого отдельный скрипт проходил по каждой найденной ссылке и извлекал содержимое карточки: имя животного, текст описания, характеристики (если они были указаны), а также ссылки на изображения.

Так как сайты приютов имеют разную структуру страниц, универсальный парсер для всех источников построить не удалось. Для каждого сайта использовалась отдельная версия скрипта с одинаковой логикой работы, но с небольшими изменениями в селекторах, структуре сбора ссылок и хранении информации. Это позволило сохранить единый подход, адаптируя его под технические особенности каждого источника. 

Результаты сохранялись в унифицированном формате таблиц. Каждая запись представляет собой отдельную карточку животного со ссылкой, номеромприюта, именем животного, видом, текстом описания, блоком характеристик и метаданными.

## Извлечение описания собаки из изображения

На данном этапе проекта была выбрана модель и подготовлен промт для извлечения текстово описания изображения животного. Было принято решение обрабатывать входные изображения собак с помощью vision-language модели LLaVA для получения описания внешнего вида животного. На ранних стадиях рассматривалась модель BLIP-2 для генерации описаний изображений. Однако в процессе тестирования было принято решение перейти на модель LLaVA, так как она генерирует более длинные и информативные тексты, в то время как BLIP-2, как правило, ограничивается короткими подписями .

Модель формирует описание на естественном языке, учитывая визуальные характеристики, такие как цвет, тип шерсти и отличительные особенности жтвотного. На вход модели подаётся изображение, а также задаётся текстовый промпт, определяющий структуру описания. После чего модель генерирует подробное описание собаки, которое используется для извлечения структурированных признаков и создания текста поста с указанием внешности животного.

## Учет стилистики и других постов приюта
Для учета стилистики изначально планировалось использовать rag для выявления постов в соцсетях по этой теме (описание животного, еженедельные отчеты, пост-прощание). Но учитывая тип данных (карточки с сайтов приютов) в этом необходимость отпала. Посты приюта загружаются в LLM и анализируются, на выходе получается список особенностей стилистики написания текстов. 

Далее для написания итового текста в ллм загружаются информация о собаке, выявленная из изображения, дополнительные характеристики от волонтера и описание стилистики написания поста. На выходе получается текст, готовый к публикации вместе с изображением, которое было загружено волонтером.
